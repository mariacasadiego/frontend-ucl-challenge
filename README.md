## Features

### Fixtures

* General fixtures listing
* Pagination
* Responsive layout

### Team View

* Browse fixtures for a selected team
* Home and away fixture statistics
* Team-specific schedule

### Matchday View

* Browse fixtures by matchday
* Quick navigation between matchdays

### Filters

* Team filter
* Matchday filter
* Country filter
* Venue filter (Home / Away)
* Active filter tags
* Clear filters action

### User Experience

* Loading state
* Error state
* Empty state
* Responsive design
* Accessible controls and navigation

## Architecture

The project follows a feature-based architecture:

```txt
src/
├── app/
├── api/
├── features/
│   └── matches/
├── shared/
│   ├── components/
│   └── styles/
└── router/
```

This structure keeps business features isolated and improves maintainability, scalability and code organization.

## Running the Project

### Install dependencies

```bash
npm install
```

### Configure environment variables

Create a `.env` file:

```env
VITE_API_URL=http://localhost:8000
```

### Start development server

```bash
npm run dev
```

### Create production build

```bash
npm run build
```

---

## Technical Decisions

### State Management

TanStack Query is used for API requests, caching and server state management.

### Styling

SCSS Modules were chosen to keep styles encapsulated and maintainable.

### Filtering Strategy

Fixture data is fetched from the API and filtered on the client side to provide a fast and responsive experience.

### Accessibility

The application includes:

* Semantic HTML
* Keyboard-friendly interactions
* ARIA attributes
* Focus-visible states
* Accessible form controls
